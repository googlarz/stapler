# v2026.423.0

> Released: 2026-04-23

## Highlights

- **Quality Flywheel + Meta-Flywheel** — Eight-pillar system that makes every agent run measurably better and lets the system improve how it organizes work, routes issues, and mines successful strategies as reusable playbooks. Fully feature-flagged per company; each pillar can be enabled independently.

## Quality Flywheel (Pillars 1–5)

- **Continuous scoring (P1)** — Every successful heartbeat run is auto-judged with a lightweight LLM rubric and stored in `run_scores`. Rubric sources cascade from issue acceptance criteria → goal criteria → generic quality rubric.
- **Self-critique gate (P2)** — Before a run finalizes as `succeeded`, agents run a self-critique pass against the success criteria. Runs that score below the company threshold transition to a new `needs_review` status and wait for human approval or a second-agent review. A panel-of-judges mode scores N times and takes the median.
- **Failure → Rule pipeline (P3)** — Low scores (`< 0.5`) and 👎 votes trigger a post-mortem agent that extracts a root-cause rule, stores it as a tagged memory, and automatically injects it into future runs on similar tasks.
- **Config change gate (P4)** — Agent config changes that touch significant keys (`systemPrompt`, `model`, `adapterType`) are gated through a pinned smoke eval suite. If the new config regresses the suite beyond threshold, the PATCH returns 409 with the eval run ID. Override with `?gate=false` when intentional.
- **Quality dashboard + drift detection (P5)** — Rolling 7d/30d/90d quality trends per agent and company. EMA-based drift detection fires a `quality.drift` activity event and optional webhook when an agent's rolling average drops more than 10% week-over-week.

## Meta-Flywheel (Pillars 6–8)

- **Organizational learning (P6)** — Every issue routing decision and goal decomposition is tracked by outcome. A Jaccard-similarity routing suggester proposes the best assignee for new issues based on historical win rates, posting a suggestion comment automatically. Goal decomposer is retrieval-augmented: it consults top-scoring past decompositions before calling the LLM.
- **Collaboration learning (P7)** — Every delegation edge (`stapler_delegate_task` / `stapler_check_delegation`) is recorded. The collaboration analyzer rolls up per-pair win rates and detects anti-patterns: ping-pong loops (A→B→A with no progress), orphan delegations (no follow-up after 4 h), and depth runaways (chain > 4 deep). Nightly job scans and emits activity events for each.
- **Workflow learning (P8)** — A nightly playbook miner clusters high-scoring runs by task similarity and tool-use pattern, writing versioned playbooks per agent. At wakeup time, the playbook injector finds the best-matching playbook and prepends its steps to the agent's context. A/B testing routes 50/50 between competing playbook versions; the winner is promoted automatically after enough runs.

## CI Improvements

- Added `ci.yml` workflow that triggers on every push to `main` — previously, direct commits to `main` bypassed all CI checks entirely.
- Pull request workflow gains a dedicated `flywheel` job that runs the four flywheel test files in isolation for clear failure attribution.
- Migration smoke check runs every PR and push against a real Postgres 15 container.
- Fixed `skill-sync.test.ts` case-insensitive key test that referenced the old `paperclipai` product name.

## Upgrade Guide

Multiple database migrations will run automatically on startup. All migrations are additive — no existing data is modified.

New tables: `run_scores`, `decomposition_outcomes`, `routing_outcomes`, `delegation_edges`, `playbooks`, `playbook_experiments`.

New columns on `agents`: `smoke_suite_id`, `self_critique_threshold`, `panel_judges`, `auto_score_runs`, `self_critique_enabled`.

New column on `heartbeat_runs`: `stdoutExcerpt` (used by the LLM judge).

New `needs_review` status on `heartbeat_runs.status` — existing code that checks terminal statuses should treat `needs_review` as terminal.
