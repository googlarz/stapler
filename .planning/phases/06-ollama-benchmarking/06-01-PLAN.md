  <action>
Create `ui/src/pages/OllamaBenchmark.tsx` as a self-contained React component with no new server endpoints.

**Ollama base URL:** Read from `DEFAULT_OLLAMA_BASE_URL` imported from `@paperclipai/adapter-ollama-local`. Add a text input at the top of the page so the user can override it (pre-filled with the default). This mirrors the pattern in `config-fields.tsx` and `model-picker.tsx`.

**Fixed benchmark prompt** (hardcode as a constant at top of file):
```
const BENCH_PROMPT = "Explain the difference between a process and a thread in one short paragraph.";
```

**Component structure:**

1. **Header** — page title "Benchmark Ollama Models" + subtitle explaining purpose.

2. **Base URL input** — labeled text input, pre-filled with `DEFAULT_OLLAMA_BASE_URL`. Changing it re-fetches the model list.

3. **Model list with checkboxes** — on mount (and when baseUrl changes), fetch `GET {baseUrl}/api/tags` with a 5s timeout. Map `body.models` to `{ name: string }[]`. Render a checkbox per model, all checked by default. If fetch fails, show an inline error with a "Retry" button. Show a spinner while loading.

4. **Run Benchmark button** — disabled when no models are checked or a run is already in progress. Label: "Run Benchmark ({N} models)" where N is the checked count.

5. **Progress indicator** — while running, show "Running... {completed}/{total}" below the button. Each model runs sequentially (not parallel) to avoid overloading Ollama.

6. **Results table** — rendered after at least one result is available (show incrementally as each model finishes). Table columns:
   - **Model** — model name (e.g. `llama3.1:8b`)
   - **Response Time** — `{N} ms` formatted with `toLocaleString()`, or "Error" in red if the request failed
   - **Output** — first 200 chars of `message.content` with trailing `…` if truncated, or error message in muted text

   Visually highlight the fastest successful model with a small "Fastest" badge (green text, no icon needed).

   Sort order: results appear in the order models were benchmarked (not re-sorted).

7. **Re-run** — after all models finish, the button label resets to "Run Benchmark ({N} models)" and clicking it clears results and runs again.

**Implementation notes:**
- Use `useEffect` with cleanup (`cancelled` flag pattern from model-picker.tsx) for the tags fetch.
- Run models sequentially: `for (const model of selectedModels) { ... }` updating state after each.
- Use `AbortController` for each chat request, store it in a ref so the user can navigate away without dangling requests.
- Convert `total_duration` (nanoseconds) → ms: `Math.round(total_duration / 1_000_000)`.
- No TanStack Query needed — plain `useState` + `useEffect` is fine for this one-shot operation.
- Import from: `lucide-react` (Loader2, AlertCircle), `@paperclipai/adapter-ollama-local`, existing Tailwind patterns.
- Do NOT import from `@tanstack/react-query` or any component that requires context (no `useCompany`, no `useDialog`).
- Use `useBreadcrumbs` from `../context/BreadcrumbContext` and set breadcrumbs to `[{ label: "Ollama Benchmark" }]` in a `useEffect`.
