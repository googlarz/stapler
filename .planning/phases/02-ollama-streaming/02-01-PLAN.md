  <name>Task 1: Stream the final text response in the tool-calling path (Path A)</name>
  <files>packages/adapters/ollama-local/src/server/execute.ts</files>
  <action>
In Path A (the agentic tool loop, starting around line 309), the final non-tool-call response is currently fetched with `stream: false` and emitted as a single `onLog` chunk. Change this so the final response uses streaming.

The tool-calling iterations that invoke tools MUST continue using `stream: false` (Ollama only returns `tool_calls` in non-streaming mode). Only the final iteration — where no tool calls are returned and the model emits plain text — should switch to `stream: true`.

Implementation approach:
1. Keep all iterations in the loop using `stream: false` until tool calls are detected.
2. When the model returns a response with NO tool calls (i.e., it is producing its final text), check `msgObj.content`. If content is non-empty, rather than emitting it as a single chunk, make a second streaming request to Ollama using the same `loopMessages` with `stream: true`, read the NDJSON response via `response.body.getReader()`, and emit per-token `onLog` calls exactly as Path B does.
3. Accumulate `assistantContent` from the streamed tokens.
4. After the stream completes, emit the `type: "done"` line as before.
5. Handle the case where the model produces no content (empty string) — skip the streaming request and go straight to the done line.
6. Wrap the streaming sub-request in the same `controller.signal` (AbortController for timeout).

The streaming sub-request body:
```typescript
const streamReq: Record<string, unknown> = {
  model,
  messages: loopMessages, // same messages that produced the non-tool-call response
  stream: true,
};
if (temperature !== undefined) streamReq.options = { temperature };
```

Read the NDJSON stream using the same pattern as Path B (lines ~488-564 in execute.ts):
- `response.body.getReader()` + TextDecoder + buffer + split("\n")
- For each line: parse JSON, extract `message.content` as token, call `onLog("stdout", JSON.stringify({type:"chunk",content:token})+"\n")`
- Accumulate into `assistantContent`
- When `done===true`: extract `prompt_eval_count` and `eval_count`

After streaming completes, emit the done line and set `toolsUsed = true`.

Do NOT change Path B — it is already correct.

IMPORTANT: Do not add tools to the streaming sub-request (Ollama streaming does not support tools). The final response in the tool loop is always a plain-text answer, never another tool call.
